# Roguelike 卡牌游戏 — 核心玩法设计 v3

> 基于 v2 设计文档的反思与精炼。砍掉了平行堆叠的系统，保留并深度融合两个核心机制。

## 设计决策总览

| 决策 | 选择 | 理由 |
|------|------|------|
| 单人 vs 多人 | **单人优先**，多人合作为增强层 | 1 人开发现实；单人体验扎实才有多人基础 |
| 核心机制 | **站位系统 + 植入体系统**，深度咬合 | 站位定义战斗空间，植入体定义构筑深度，两者交叉产生涌现玩法 |
| 遗物系统 | **砍掉**，职能由植入体 + 事件奖励吸收 | 遗物和植入体边界模糊，合并后设计空间更清晰 |
| 入侵系统 | **降级为卡牌效果**（不再是独立系统） | "控制敌人"保留为黑客的高阶卡牌效果，不再有独立进度条和资源 |
| 改造/零件 | **砍掉**，只保留金币单一货币 | 双货币没有产生有意义的取舍 |
| 协同链 | **砍掉独立系统**，部分思路融入多人增强层 | 单人优先策略下，协同链无法成为核心；多人时用轻量"共鸣"替代 |
| 适应性 AI | **砍掉**，改为多阶段行为模式 | 固定但丰富的敌人行为 > 动态适应（见下文详述） |
| 动态环境 | **保留 5 种**，仅"规则改变"类 | 砍掉剥夺信息/纯随机类环境 |
| PvP | **v1 不做** | 资源集中在 PvE 体验 |

## 一、核心循环

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   选择路径 → 进入战斗 → 获得奖励 → 强化构筑 ──┐     │
│       ↑                                        │     │
│       └────────────────────────────────────────┘     │
│                                                     │
│   强化构筑 = 选新卡 / 升级卡 / 更换植入体 / 商店    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**每次"强化构筑"的决策都必须同时考虑两个维度：**
1. 我的牌组方向（攻击流？防御流？控制流？）
2. 我的站位策略（常驻前排？常驻后排？频繁换位？）

植入体是连接这两个维度的铰链——它同时改变你的出牌规则和站位收益。

## 二、站位系统（重新设计）

### 2.1 基础规则

```
┌───────────────────────────┐
│   【后排敌人】              │
│    受前排遮挡              │
│                           │
│   【前排敌人】              │
│    优先被攻击              │
│                           │
│   ══════ 战线 ══════      │
│                           │
│   【前排】你可以站在这里    │
│    · 近战牌满额伤害        │
│    · 每次获得护甲时额外 +3  │
│    · 承受所有单体攻击      │
│                           │
│   【后排】或者站在这里      │
│    · 远程牌伤害 +20%       │
│    · 近战牌不可使用 ← 关键  │
│    · 单体攻击不会打到你     │
│      （除非前排无人/穿透）  │
│                           │
└───────────────────────────┘
```

### 2.2 与 v2 的关键区别

**改动一：后排不能打近战牌（而非 -25% 伤害）**

v2 的设计是后排打近战牌伤害 -25%。这太温和了——玩家永远待后排，-25% 很容易被其他加成抹平。

新规则：**后排时，近战牌变灰不可使用。** 这迫使你在构筑时就决定：我走近战多还是远程多？如果近战多，我必须常驻前排承受伤害；如果远程多，我在后排安全但放弃了护甲加成。

近战/远程不是新属性，而是对现有卡牌的分类标记：
- ⚔️ **近战**：直接伤害类（斩击、重锤、护盾猛击）
- 🎯 **远程**：投射/能量类（射击、灵能冲击、数据脉冲）
- ⚙️ **通用**：技能/状态类（护甲、抽牌、Buff/Debuff）— 不受站位限制

每个职业的卡池有不同的近战/远程比例，这自然定义了职业的"推荐站位"，但植入体可以颠覆这个比例。

**改动二：换位有成本**

每回合可以使用一次"换位"，消耗 **1 点能量**（不是免费的）。

这意味着：
- 换位 = 少打一张 1 费牌。是否值得？
- 某些回合你会被"卡"在错误的位置（手里全是近战牌但你在后排），这产生了张力。
- 专门用于换位的卡牌（0 费换位、换位后抽牌）变得有构筑价值。

**改动三：前排风险真实存在**

敌人的单体攻击**只打前排**。不是"优先"前排——是**只打**前排。后排绝对安全（除非遇到穿透敌人或前排无人）。

这让站位选择变成了一个清晰的赌注：
- 前排 = 更强的输出/防御能力，但要吃所有伤害
- 后排 = 安全，但输出手段受限（只有远程和通用牌）

### 2.3 远程加成：第一张牌，而非全部

后排远程加成不是静态倍率，而是**每回合第一张远程牌 +20%**，之后恢复 1.0×。

这一改动来自 Brainstorm 审查：静态 +20% 会让"无脑待后排打远程"成为最优解，消解换位的意义。新规则：
- 第一张远程牌 → +20%（鼓励偶尔换到后排打"开幕炮"）
- 后续远程牌 → 无加成（不鼓励整回合蹲在后排）

这让远程牌的最优利用需要**有意识的换位节奏**，而非被动待后排。

### 2.4 位置反应式卡牌（Brainstorm 高优先级实现）

部分卡牌有两套截然不同的效果，取决于出牌时玩家的站位：

```
【相位突进】（先锋）1 费
├ 前排：对目标造成 8 点伤害，然后退入后排
└ 后排：移入前排，获得 8 点护甲
```

设计原则：
- **没有"弱效果"版本**——前后排效果是等价的，只是适应不同的战斗节奏
- 卡牌描述同时展示两套效果，玩家在出牌前可看到当前位置会触发哪条
- 4 个职业各有代表性的位置反应牌：先锋 `phase_charge`、灵能者 `psi_lance`、黑客 `ghost_probe`、共生体 `bio_adapt`

### 2.5 站位与敌人设计的联动

敌人也分前后排，并且敌人的站位创造了战术问题：

| 敌人配置 | 给玩家的战术问题 |
|---------|-----------------|
| 前排肉盾 + 后排输出 | 你是先清前排（快但后排持续输出），还是用穿透/拉扯绕过前排？ |
| 全前排近战群 | 后排远程构筑的天堂——但如果你是近战 build，就必须站前排对撞 |
| 后排治疗 + 后排 Boss | 你必须想办法够到后排，否则 Boss 永远被奶满 |
| 单个穿透敌人 | 你的后排不再安全，必须决定是否换到前排用护甲硬吃 |

**设计原则：每种敌人配置都应该让"站前排"和"站后排"各有道理，没有无脑最优解。**

### 2.6 穿透与拉扯（关键词定义）

部分卡牌和敌人能力引用了"穿透"和"拉扯"，定义如下：

| 关键词 | 定义 | 设计意图 |
|--------|------|---------|
| **穿透** | 无视前排遮挡，直接命中后排目标。对护甲正常结算。 | 打破"后排=安全"的假设，给纯后排 build 制造压力 |
| **拉扯** | 将后排目标强制移到前排（敌人或玩家均可被拉扯）。被拉扯的单位下回合可正常换位。 | 破坏对方站位规划，创造窗口期 |

- 穿透是**攻击属性**：标记在卡牌或敌人技能上，使该次攻击忽略前排遮挡
- 拉扯是**控制效果**：标记在卡牌上，强制改变目标站位
- 两者可组合：先拉扯后排敌人到前排，再用近战牌全额伤害

## 三、植入体系统（重新设计）

### 3.1 核心原则

**植入体改变规则，不改变数值。**

以下不是植入体：
- ❌ "最大 HP +10"（这是数值，用卡牌升级或事件做）
- ❌ "护甲保留 50%"（这是被动效果，原来遗物的职能）
- ❌ "每回合抽牌 +1"（数值）

以下才是植入体：
- ✅ "近战牌现在可以在后排使用，但每张消耗 1 HP"（改变站位规则）
- ✅ "你的护甲不再被攻击消耗，而是在回合结束时全部失去"（改变护甲结算规则）
- ✅ "换位不再消耗能量，但每次换位你失去所有护甲"（改变换位规则）

### 3.2 插槽设计：2 个，不是 3 个

砍掉一个插槽。3 个插槽 × 多选项 = 排列组合爆炸，测试不完。

| 插槽 | 影响 | 设计意图 |
|------|------|---------|
| 🧠 **神经插槽** | 改变出牌/能量/抽牌的**规则** | 决定你"怎么打牌" |
| ⚡ **核心插槽** | 改变职业核心机制 + 站位交互的**规则** | 决定你"打什么 build" |

注意：没有"躯体插槽"。HP/护甲相关的被动改造通过事件奖励和卡牌升级来实现，不需要专门的插槽。

### 3.3 植入体设计：每个都是"build 定义器"

#### 神经插槽（通用，所有职业共享）

| 植入体 | 规则改变 | 它定义的 build |
|--------|---------|---------------|
| **标准神经接口** | 每回合抽 5 牌，3 能量。（默认） | 基础平衡型 |
| **过载回路** | 每回合抽 5 牌，4 能量。但每回合结束时，未使用的能量转化为等量伤害打在自己身上。 | "用光每一点能量"的精密规划型——鼓励你构筑恰好用完能量的牌组 |
| **脉冲处理器** | 每回合抽 3 牌，2 能量。但你每打出一张牌，立刻抽一张牌。 | 低费连锁型——大量 0-1 费牌的引擎构筑 |
| **预知模块** | 每回合抽 4 牌，3 能量。每 2 回合（偶数回合），你可以查看抽牌堆顶 2 张牌，并选择其中 1 张加入手牌（另一张放回底部）。 | 精准控制型——适合关键牌 timing 很重要的 build，但预知频率受限避免过度控制 |
| **战场接口** | 每回合抽 5 牌，3 能量。换位不再消耗能量，且每次换位后你的下一张牌费用 -1。 | 频繁换位型——为"前后排都要站"的混合 build 而生 |

#### 核心插槽（职业专属，以先锋为例）

先锋的默认核心机制是**超载 (Overcharge)**：打出攻击牌累积超载层数，消耗超载释放大伤害。

| 植入体 | 规则改变 | 它定义的 build |
|--------|---------|---------------|
| **超载核心**（默认） | 标准超载机制 | 蓄力爆发型先锋 |
| **铁壁核心** | 取消超载。改为：你在前排时，每回合开始获得等同于"你上回合打出的近战牌数量 ×4"的护甲。 | 前排坦克先锋——近战牌越多，护甲越厚，牢牢钉在前排 |
| **狂战核心** | 取消超载。改为：每打出一张攻击牌（近战或远程），本回合后续所有攻击牌伤害 +2（可叠加）。在后排时每张远程攻击牌的叠加量翻倍（+4 而非 +2）。 | 连续攻击型——后排加成更高但只能用远程牌叠加，鼓励构筑大量远程攻击牌的先锋，完全颠覆默认近战路线 |
| **指挥核心** | 取消超载。改为：你在前排时，你打出的每张通用牌额外给自己 3 点护甲。你在后排时，你打出的每张通用牌额外抽 1 张牌。 | 辅助/灵活型先锋——站位决定你是坦还是辅助 |

### 3.4 植入体与站位的咬合

**这是整个设计的核心——植入体重新定义了站位的收益函数。**

没有植入体时，每个职业有"天然最优站位"（先锋站前排，灵能者站后排）。装上植入体后，最优站位可能完全反转：

| 场景 | 站位决策 |
|------|---------|
| 先锋 + 超载核心（默认） | 前排（近战牌多，需要近距离） |
| 先锋 + 狂战核心 | 后排更优（远程牌叠加翻倍），但需要改造牌组为远程主导 |
| 先锋 + 铁壁核心 | 锁死前排（离开就没有护甲生成） |
| 先锋 + 指挥核心 + 战场接口 | 频繁换位（前排攒甲，后排抽牌，每回合切换最大化收益） |

**这就是"两个系统深度咬合"的效果：植入体的选择改变了站位的意义，站位的选择又影响了你需要什么卡牌，卡牌的选择又反过来决定了哪些植入体最优。三角互锁。**

### 3.5 获取与更换

- **获取**：精英战掉落（二选一）、Boss 掉落（三选一）、商店高价购买、特殊事件
- **更换限制**：每幕（Act）只能更换 1 次植入体。更换时旧植入体销毁。
- 这意味着你在第一幕选的植入体大概率陪你到底——选择有重量。

### 设计原则

1. **核心机制**（一个独有的战斗资源/状态）

## 四·五、原型核心系统（Prototype Core）

> Brainstorm 实现：解决"零信息开局选机制"问题，改为观察行为后自动激活。

### 什么是原型核心

原型核心是每场战斗的动态 buff 系统。前 3 回合，系统在后台**观察**玩家的出牌习惯，第 3 回合结束时自动激活匹配的协议：

| 观察指标 | 激活协议 | 效果 |
|---------|---------|------|
| 从前排打近战牌居多 | **前压协议** | 每回合第一张近战牌额外获得 +1 超载层 |
| 主动换位次数居多 | **战术协议** | 换位后下一张牌费用 -1 |
| 从后排打远程居多 | **远程压制协议** | 每回合第一张远程牌额外 +3 伤害 |

**设计意图：** 避免"零信息情况下选机制"的困境。改为让玩家**自然地打前 3 回合**，系统根据实际行为匹配最合适的协议并展示激活提示。不同玩家在同一职业下会自然激活不同协议，形成隐式的 build 分化。

### 卡牌分支升级（Branch Upgrades）

每张可升级的卡牌提供 **A/B 两条分支**，而非固定单一升级路径：



升级分支在休息点/精英奖励时展示，玩家当即选择。这意味着：
- 两份相同卡牌可以走完全不同的分支
- build 方向定型后，升级选择成为强化 build 的工具，而非"必吃"的数值提升

### 敌人位置关键词（Enemy Position Keywords）

部分敌人技能带有**位置关键词**，触发特殊规则：

| 关键词 | 效果 | 设计意图 |
|--------|------|---------|
| **lock（锁定）** | 攻击命中时，对目标施加「根植」状态（1 回合无法换位） | 强制玩家在错误位置结算，考验手牌适应能力 |
| **breakCover（破甲）** | 前排目标受到攻击时，后排玩家也受到 50% 溅射伤害 | 打破"后排绝对安全"的假设，迫使后排玩家表现出正面价值 |

这两个关键词使敌人对站位产生主动威胁：lock 迫使你在错误位置应对，breakCover 让后排不再是免费的安全区。


## 五、敌人设计原则

### 5.1 不做适应性 AI，做"行为阶段"

每个敌人有 2-3 个**行为阶段**，由 HP 阈值触发切换：

```
重装哨兵 (HP: 40)

阶段 1 (HP > 50%):
  意图循环: 攻击 8 → 防御 6 → 攻击 12 → 防御 6

阶段 2 (HP ≤ 50%):
  意图循环: 攻击 15 → 攻击 15 → 充能(下回合攻击全体 20)
  
  切换时: 释放一次"警戒脉冲"（全体 5 伤害）
```

**为什么这比适应性 AI 好：**
- 玩家可以**学习**每个敌人的行为——"哨兵半血后会狂暴，我要在它半血前攒好护甲"
- 行为切换是**可预测但有压力的**——你知道它要变，但你不一定准备好了
- 设计成本低，测试成本低，玩家理解成本低

### 5.2 敌人组合创造战术问题

单个敌人不需要很复杂。复杂度来自**组合**：

| 组合 | 战术问题 | 站位影响 |
|------|---------|---------|
| 前排盾兵 ×2 + 后排治疗无人机 | 必须绕过/清掉前排才能杀治疗，否则无限回复 | 穿透型 build 价值凸显 |
| 前排自爆虫 ×3（死亡时爆炸 8 伤害） | 一个个杀会吃 3 次爆炸，但你没有 AOE... | 后排更安全（远离爆炸），但你需要远程牌逐个处理 |
| 后排狙击手（无视前排，直接打你） + 前排普通兵 | 后排不再安全——你必须决定是换前排用护甲硬吃狙击，还是快速穿透杀掉狙击手 | 打破"后排=安全"的假设，制造换位压力 |
| 前排反击骑士（受到近战攻击时反弹 5 伤害） + 后排强化者 | 近战 build 被惩罚，需要切换到远程手段或忍受反弹 | 迫使近战 build 临时退后排 |

### 5.3 精英和 Boss

**精英**：比普通敌人多一个"特殊规则"（类似环境效果但只影响这场战斗）

示例：
- **磁力守卫**（精英）：每 2 回合强制将你换位一次。你必须围绕被动换位来规划出牌。
- **镜像猎手**（精英）：复制你上回合打出牌数最多的类型（近战/远程/通用），对你使用。

**Boss**：多阶段 + 专属战场规则 + 独特的站位挑战

示例（第一幕 Boss）：

```
██ 废墟守护者 Derelict Guardian ██
HP: 250 | 占据前后排两个位置（巨型敌人）

阶段 1 — 巡逻模式 (HP > 60%)
  前排形态: 近战猛击 18 → 护甲 15 → 近战横扫(全体 10)
  ※ 每 3 回合切换到后排形态
  后排形态: 远程射击 12 → 部署前排护卫犬 → 远程散射(全体 8)
  ※ 每 3 回合切换到前排形态

阶段 2 — 锁定模式 (HP ≤ 60%)
  锁定在前排，不再切换。
  意图循环: 充能 → 充能 → 毁灭光束(全体 30，但充能回合不攻击)
  ※ 2 个充能回合是你的输出窗口

阶段 3 — 崩溃模式 (HP ≤ 25%)
  核心暴露（受到的伤害 +50%）
  意图: 每回合自爆倒计时 -1（从 3 开始）
  ※ DPS 检测——3 回合内必须杀死，否则全体 99 伤害
```

**站位互动**：
- 阶段 1 前排形态——你站前排会吃 18 近战，站后排安全但只能远程输出
- 阶段 1 后排形态——它在后排远程打你，前后排区别不大，但它会刷护卫犬挡前排
- 阶段 2——充能回合站前排输出最大化（近战牌满额），光束回合退后排保命
- 阶段 3——全力输出，站位不重要了，纯 DPS 竞速

## 六、环境效果（精简版）

只保留 5 种，全部是"改变规则"类型。**没有"剥夺信息"或"纯随机"类型。**

| 环境 | 规则改变 | 对站位策略的影响 |
|------|---------|-----------------|
| ⚡ **等离子风暴** | 所有攻击牌伤害 +3，所有通用牌费用 +1 | 纯攻击 build 受益；依赖通用牌的黑客/灵能者被压制 |
| 🔄 **重力异常** | 换位不消耗能量。但每次换位后你失去所有护甲 | 鼓励频繁换位，但惩罚叠甲型 build——选择有代价 |
| ❄️ **低温区域** | 每回合结束时，你手牌中费用最高的 1 张牌被"冻结"（下回合不可用） | 逼迫你用掉大牌或构筑低费牌组——和脉冲处理器植入体配合极佳 |
| 🔥 **过热区域** | 前排的单位（包括敌人）每回合额外受到 3 点伤害 | 前排更危险，后排远程 build 天然优势。但敌人前排也在掉血——近战速攻 build 可能反而更快清场 |
| 💾 **数据节点** | 每打出第 3 张牌时，抽 1 张牌 | 奖励大量出牌的引擎 build——和编织核心（黑客）、脉冲处理器极度契合 |

**设计原则：每种环境都应该让某些 build 变强、某些变弱，但没有任何 build 被完全废掉。** 环境是调味料，不是惩罚。

## 七、单局流程（单人模式）

### 7.1 结构

```
第一幕 — 7 层节点（3 战斗 + 1 精英 + 1 商店 + 1 事件 + 1 休息）→ Boss

第二幕 — 8 层节点 → Boss

第三幕 — 9 层节点 → 最终 Boss
```

### 7.2 节点类型

| 节点 | 功能 |
|------|------|
| ⚔️ 普通战斗 | 打完选 1/3 张新卡 |
| 💀 精英战 | 更难，打完选 1/3 张新卡 + 获得一个植入体（二选一） |
| 🏕️ 休息点 | 回复 30% HP **或** 升级一张牌 **或** 查看下一层全部节点信息 |
| 🛒 商店 | 买卡、买药水、买植入体（高价）、移除卡牌 |
| ❓ 事件 | 叙事选择，获得特殊奖励/惩罚（如"永久 +5 HP 但在前排时受到伤害 +2"） |
| 🔮 Boss | 每幕末尾，击败后获得核心插槽植入体（三选一） |

### 7.3 奖励节奏

#### 植入体获取节点

- 第一幕精英（约第 3-4 层）：获得第一个**神经插槽**植入体
- 第一幕 Boss：获得第一个**核心插槽**植入体 ← 这里你的 build 方向定型
- 后续幕的精英/Boss：可以更换（每幕限 1 次），微调 build

**关键设计：核心插槽植入体在第一幕 Boss 后才获得。** 这意味着第一幕你用默认核心机制打，在这个过程中你根据获得的卡牌来判断哪个核心植入体最配你当前牌组。这避免了"一开始就选植入体但不知道自己会拿到什么牌"的信息不足问题。

#### 卡牌升级

- **升级途径**：休息点（消耗休息机会）、商店（花费金币）、特定事件
- **升级效果**：每张卡只能升级一次。升级后卡牌获得"+"后缀（如 `斩击+`），数值提升约 30% 或附加次要效果
- **设计意图**：升级是中后期主要的成长轴之一——当牌组方向已定型，"打磨现有牌"比"加新牌"更有价值

#### 货币系统

- **单一货币：信用币（Credits）**
- 获取：战斗胜利（普通战 15-25，精英 30-50）、事件奖励、卖卡
- 用途：商店买卡（50-150）、买药水（30-60）、买植入体（200-300，高价）、移除卡牌（75，每次递增 25）、升级卡牌（80）
- **设计意图**：金币是"通用机会成本"——你想做的事永远比钱多，迫使取舍

#### 药水系统

- **携带上限：2 瓶**（多人模式 3 瓶）
- 获取：精英战必掉 1 瓶、商店购买、事件奖励
- 药水为一次性消耗品，在战斗中任意时刻使用，不消耗能量
- 示例药水：治疗药剂（回复 30% HP）、能量药剂（本回合 +2 能量）、换位药剂（免费换位并抽 2 牌）、爆发药剂（本回合所有攻击 +50% 伤害）
- **设计意图**：药水是"保险"和"爆发窗口"——给关键回合一个额外选项，弥补手牌运气差的场次

## 八、多人合作增强层（概述）

> **注意**：本节仅为多人设计的概述。详细的仇恨公式、站位规则、缩放测试数据等应在独立的 `multiplayer-design.md` 中展开。当前文档聚焦单人核心玩法。

### 8.1 设计原则

多人模式不应该要求新系统，而是让现有系统在多人环境下自然产生新的互动。

### 8.2 多人站位

2-4 人合作时，前排和后排各可以站多人：

```
【前排】先锋 + 共生体     ← 两人分摊伤害（仇恨值决定谁被打）
【后排】灵能者 + 黑客      ← 安全输出
```

**仇恨规则（仅多人模式）**：
- 前排有多人时，敌人攻击本回合造成伤害最高的前排玩家
- "嘲讽"卡牌可以强制敌人攻击自己

### 8.3 轻量协同（替代协同链）

不设计专门的"链接卡"系统。协同通过自然的系统交互实现：

| 协同方式 | 举例 | 为什么不需要专门系统 |
|---------|------|---------------------|
| 站位配合 | 先锋站前排扛伤害，灵能者安心后排输出 | 站位系统天然产生 |
| 目标集火 | 黑客用渗透核心暴露后排敌人，灵能者打暴露目标 +50% 伤害 | 状态效果天然产生 |
| 换位配合 | 共生体用裂变核心前后排频繁切换，空出前排位置给需要短暂前排的灵能者 | 换位规则天然产生 |
| 药水投掷 | 某些药水可以选择给自己用或"投给"队友 | 简单的目标选择 UI |
| 倒下救援 | 玩家 HP 归零进入濒死（3 回合），队友用治疗牌拉起 | 复用现有治疗卡 |

**核心理念：好的多人体验不需要专门的多人机制。** 如果单人玩法的系统足够丰富，多人时它们自然会产生配合。

### 8.4 多人数值缩放

| 人数 | 敌人 HP 倍率 | 敌人数量 | 玩家 HP 调整 |
|------|-------------|---------|-------------|
| 1 人 | ×1.0 | 基准 | 基准 |
| 2 人 | ×1.8 | +1 敌人/波 | 基准 |
| 3 人 | ×2.7 | +1 敌人/波 | -5% HP |
| 4 人 | ×3.8 | +2 敌人/波 | -10% HP |

> **缩放说明**：人均 HP 负担分别为 1人 100%、2人 90%、3人 85.5%、4人 85.5%。多人站位分摊伤害和协同效果会抵消部分难度，因此人均负担递减但幅度收敛，确保 4 人不会比 2 人显著更轻松。

## 九、系统交互矩阵

验证所有系统之间的关系——每个系统是否让其他系统更有趣：

| | 站位 | 植入体 | 卡牌构筑 | 敌人行为阶段 | 环境效果 |
|---|---|---|---|---|---|
| **站位** | — | 植入体改变站位收益 ✅ | 近战/远程限制影响选卡 ✅ | 敌人组合迫使换位 ✅ | 某些环境改变站位代价 ✅ |
| **植入体** | 决定最优站位 ✅ | — | 定义 build 方向→选卡方向 ✅ | 某些植入体针对性克制敌人类型 ✅ | 某些植入体和环境极度契合 ✅ |
| **卡牌构筑** | 近战/远程比例决定站位倾向 ✅ | 牌组方向决定植入体选择 ✅ | — | 需要针对敌人弱点选卡 ✅ | 环境改变某些卡的价值 ✅ |
| **敌人行为阶段** | 迫使玩家调整站位 ✅ | 提供获取植入体的节点 ✅ | 提供选卡奖励 ✅ | — | 环境也影响敌人 ✅ |
| **环境效果** | 改变前后排的风险收益 ✅ | 和植入体产生 combo ✅ | 改变卡牌的相对价值 ✅ | 影响敌人（过热区敌人前排也掉血） ✅ | — |

**25 个交叉格全部有意义的交互 ✅** — 没有孤立系统。

## 十、与 v2 的删减对照

| v2 系统 | v3 处理 | 理由 |
|---------|---------|------|
| 前后排站位 | ✅ 保留并强化 | 核心机制 |
| 协同链 | ❌ 砍掉 | 多人自然协同替代 |
| 入侵系统（独立） | ❌ 砍掉独立系统 | 浓缩为黑客的一个植入体 |
| 改造/零件 | ❌ 砍掉 | 双货币无意义取舍 |
| 植入体（3 槽） | ✅ 保留，减为 2 槽 | 控制组合爆炸 |
| 动态环境（10 种） | ✅ 保留，减为 5 种 | 只留"规则改变"类 |
| 适应性 AI | ❌ 砍掉 | 多阶段固定行为替代 |
| 遗物系统 | ❌ 砍掉 | 职能被植入体+事件吸收 |
| PvP | ❌ 推迟到 v2+ | 资源集中 |
| 多人合作 | ⚠️ 降级为增强层 | 单人优先 |

**净减少：6 个独立系统 → 0 个新增系统。** 所有复杂度来自 2 个核心系统（站位 + 植入体）的深度交叉。
