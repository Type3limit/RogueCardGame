# 🎴 Roguelike 卡牌游戏开发规划 v2

## 项目概述

**项目代号**: RogueCardGame  
**类型**: 多人合作 Roguelike 卡牌构筑 (Co-op Deckbuilder Roguelike)  
**核心参考**: 杀戮尖塔 (Slay the Spire) + 前后排站位 + 五大创新机制  
**风格**: 科幻未来 × 二次元  
**目标平台**: PC (Steam)  
**商业模式**: 买断制  
**引擎**: Godot 4.x + C#  
**开发规模**: 1 人独立开发  
**多人模式**: 2-4 人在线合作 PvE + PvP 对战  

---

## ★ 核心差异化：五大创新机制

> 以下是本项目与杀戮尖塔等同类游戏的**根本区别**。这些机制互相交织，形成一个独特的游戏体验。

### ★1. 前后排站位系统 (Formation System)

**竞品分析**: Roguebook 有双人换位但仅限2角色；Monster Train 是三层塔防而非RPG站位；HELLCARD 有网格但不是前后排仇恨。**我们的前后排系统结合多人合作和仇恨机制，是目前市场的空白。**

#### 核心规则

```
┌─────────── 战场布局 ───────────┐
│                                 │
│   【后排敌人区】                  │
│    远程敌人 / Boss 本体          │
│                                 │
│   【前排敌人区】                  │
│    近战敌人 / 护卫 / 召唤物      │
│                                 │
│   ─────── 分 界 线 ───────      │
│                                 │
│   【前排玩家区】                  │
│    坦克 / 近战角色               │
│                                 │
│   【后排玩家区】                  │
│    远程 / 辅助角色               │
│                                 │
└─────────────────────────────────┘
```

**仇恨 (Aggro) 机制**:
- 敌人**单体攻击**优先攻击**前排**玩家 (除非有特殊技能如"狙击"无视站位)
- 前排玩家全部倒下后，后排直接暴露
- **仇恨值**: 每个玩家有隐性仇恨值，受攻击/治疗/Buff行为影响。敌人攻击同排中仇恨值最高的目标
- **嘲讽 (Taunt)**: 某些卡牌/遗物可强制敌人攻击自己
- **AOE攻击**: 分为"全体"、"前排"、"后排"、"单列"等不同范围

**位置切换**:
- 玩家每回合可以免费切换一次前后排 (消耗该回合的"移动行动")
- 某些卡牌可以强制切换自己或敌人位置（"拉扯"/"击退"）
- 位置影响卡牌效果:
  - 前排打出近战攻击牌：满额伤害
  - 后排打出近战攻击牌：伤害 -25% (除非目标被拉到前排)
  - 前排打出远程攻击牌：正常伤害
  - 后排打出远程攻击牌：伤害 +15% (后排增幅)
  - 前排获得护甲：额外 +2 (前排坚守加成)
  - 后排获得护甲：正常值

**职业与站位的天然搭配**:
| 职业 | 推荐站位 | 理由 |
|------|---------|------|
| 先锋 | 前排 | 高 HP + 护甲机制，天然坦克 |
| 灵能者 | 后排 | 低 HP，远程控制/输出 |
| 黑客 | 后排 | 需要安全环境构建协议栈 |
| 共生体 | 灵活 | 自残机制使前后排各有风险/收益 |

**敌人站位策略**:
- 敌人也分前后排，后排敌人被前排"遮挡"
- 某些敌人组合：前排肉盾 + 后排治疗/增益
- 要高效击杀，需要使用"穿透"、"拉扯"、或AOE清除前排

### ★2. 协同链系统 (Synergy Link) — 多人独创

**这是本游戏最核心的多人差异化机制。** 杀戮尖塔2的多人只是"同时打牌"，我们要让玩家之间真正产生**有意义的协作**。

#### 核心概念: 链接卡 (Link Card)

某些卡牌是"**链接卡**"（卡面有特殊的链接标记 🔗），打出后不会立刻完全结算，而是进入"**链接等待状态**"。如果**另一位玩家**在同回合内打出对应的"**响应卡**"，则触发**强化版组合效果**。

#### 链接类型

| 链接类型 | 触发条件 | 效果示例 |
|---------|---------|---------|
| **攻击链** | A 打出标记攻击 → B 打出任意攻击 | B 的攻击伤害翻倍 |
| **防御链** | A 给 B 施加护盾 → B 打出防御牌 | 双方都获得额外护甲 |
| **增幅链** | A 打出"目标锁定" → B 打出攻击 | B 的攻击无视护甲 |
| **连锁链** | A 施加Debuff → B 打出对应属性攻击 | 触发"引爆"效果造成额外伤害 |
| **治疗链** | A 打出治疗牌 → B 也打出治疗牌 | 双方回复量 ×1.5 |

#### 示例卡牌

```
┌──────────────────────────┐
│ 🔗 战术标记 (1能量)        │
│ [先锋专属·链接卡]          │
│                           │
│ 标记一个敌人。              │
│ 如果队友本回合攻击该敌人，  │
│ 造成额外 8 点伤害          │
│ 并使目标获得 2 层易伤。     │
│                           │
│ (无队友响应: 仅造成 4 点   │
│  伤害和 1 层易伤)          │
└──────────────────────────┘
```

```
┌──────────────────────────┐
│ 🔗 数据共享 (1能量)        │
│ [黑客专属·链接卡]          │
│                           │
│ 选择一位队友，复制其手牌   │
│ 中最高费用的一张牌到你的   │
│ 手中（本回合消耗后消失）。  │
│                           │
│ 链接响应: 如果队友同时打出 │
│ 一张牌，你复制的牌费用-1。 │
└──────────────────────────┘
```

#### 链接系统的设计哲学
1. **链接卡即使没有队友响应也有基础效果**（不能让单人玩家感觉废卡）
2. **链接成功是锦上添花**，不是必须条件
3. **语音/文字沟通成为游戏体验的一部分**（"我下一张打标记，你准备输出！"）
4. **PvP 模式中链接卡在双人组队时同样可用**

### ★3. 入侵/改造系统 (Hack & Salvage)

科幻主题下的独特机制，给予玩家"**非暴力解决方案**"。

#### 入侵 (Hack)

- 黑客职业的核心扩展能力，其他职业可通过无色卡/遗物获得
- 对敌人使用"入侵"类卡牌，累积**入侵进度条**
- 进度满后，敌人被**暂时控制** 1-2 回合，为你方战斗
- 被入侵的敌人使用它原有的 AI 意图攻击其他敌人
- Boss 不可入侵，但可以入侵 Boss 的召唤物
- 入侵结束后敌人恢复正常但获得"系统混乱"debuff（降低攻击力）

**示例卡牌**:
- 💻 **后门植入** (1能量) - 对目标敌人施加 3 层入侵进度 (需要 10 层触发控制)
- 💻 **暴力破解** (2能量) - 施加 6 层入侵进度，但你失去 1 点能量
- 💻 **木马程序** (1能量) - 施加 2 层入侵进度，如果目标本回合行动，额外施加 3 层

#### 改造 (Salvage)

- 击败敌人后有概率获得**零件 (Parts)**
- 零件可以在休息点或专门的"**改造站**"（新房间类型）使用
- 改造选项:
  - 🔧 **卡牌改造**: 消耗零件为一张卡牌附加额外效果（类似于"升级+"）
  - 🔧 **植入体制造**: 消耗大量零件制造植入体模块（见机制4）
  - 🔧 **临时装备**: 消耗零件获得一件持续 3 场战斗的临时增益装备

**改造给予玩家额外的资源维度**，不再只有"金币"一种货币，增加了决策深度：
- 金币用于商店购买成品
- 零件用于自定义改造

### ★4. 模块化植入体系统 (Cybernetic Implant System)

**这是突破"固定职业"框架的核心系统**。

#### 概念

每个角色有 **3 个植入体插槽**:
- 🧠 **神经插槽** (Neural) — 影响抽牌/能量/回合机制
- 💪 **躯体插槽** (Somatic) — 影响 HP/护甲/站位相关
- ⚡ **核心插槽** (Core) — 影响职业核心机制的变体

每个职业**初始自带一组默认植入体**（定义了职业的基础机制），但在游戏过程中可以**替换**为其他植入体，从根本上改变打法。

#### 植入体示例

**神经插槽 (Neural)**:
| 植入体 | 效果 | 改变了什么 |
|--------|------|-----------|
| 标准神经接口 (默认) | 每回合抽 5 张牌, 3 点能量 | 基础设置 |
| 超线程处理器 | 每回合抽 7 张牌, 但只有 2 点能量 | 适合低费海量出牌的构筑 |
| 量子预测模块 | 每回合抽 4 张牌, 但可以看到抽牌堆顶 3 张牌的顺序 | 适合精准控制的构筑 |
| 过载回路 | 每回合抽 5 张牌, 3 能量, 但每打出一张牌受到 1 点伤害 | 高风险高回报 |

**躯体插槽 (Somatic)**:
| 植入体 | 效果 | 改变了什么 |
|--------|------|-----------|
| 标准动力骨架 (默认) | 正常 HP, 正常护甲机制 | 基础设置 |
| 纳米再生体 | 最大 HP -20%, 但每回合回复 3 HP | 适合持久战构筑 |
| 相位装甲 | 护甲不会在回合结束时消失 (保留 50%) | 适合叠甲构筑 |
| 轻量化骨架 | HP -30%, 但切换站位不消耗行动, 后排伤害加成 +25% | 适合机动后排输出 |

**核心插槽 (Core) — 以先锋为例**:
| 植入体 | 效果 | 改变了什么 |
|--------|------|-----------|
| 超载核心 (默认) | 标准超载机制 | 先锋默认玩法 |
| 狂战核心 | 取消超载机制，改为: 每打出一张攻击牌，下一张攻击牌伤害 +3 | 从蓄力爆发变成持续输出 |
| 守护核心 | 取消超载机制，改为: 每获得护甲时，对所有攻击你的敌人反弹 2 点伤害 | 从输出变成反伤坦 |
| 指挥核心 | 超载层数现在也为**队友**提供 1 点护甲/层 | 从自我爆发变成团队增益 |

#### 获取方式
- 改造站 (用零件制造)
- 精英战/Boss 掉落
- 特殊事件奖励
- 商店（高价出售）

#### 设计哲学
- 植入体让**同一个职业可以有截然不同的玩法**
- 与遗物的区别: 遗物是"加法"（给你多一个被动），植入体是"乘法"（改变你的核心规则）
- 每局只能更换有限次数的植入体（防止频繁切换无成本）

### ★5. 动态战场环境 (Dynamic Battlefield)

每场战斗不再只是"空房间里打怪"，战场本身成为一个战略变量。

#### 环境效果

每个房间随机附带 0-2 个环境效果（精英/Boss 房间固定有特殊环境）:

| 环境 | 效果 | 策略影响 |
|------|------|---------|
| ⚡ **EMP 干扰区** | 所有"编译"/"协议栈"类效果费用+1 | 黑客弱化，需要调整策略 |
| ☢️ **辐射泄漏** | 每回合所有单位(含敌人)受到 2 点伤害 | 共生体的再生优势凸显 |
| 🌀 **重力异常** | 前后排切换不消耗行动，但护甲效果减半 | 鼓励频繁换位的打法 |
| 🔥 **等离子风暴** | 攻击牌伤害 +3，但技能牌费用 +1 | 纯攻击构筑受益 |
| ❄️ **冷冻舱** | 每回合结束时，手牌中最右侧的牌被"冻结"(下回合不可使用) | 手牌管理变得重要 |
| 🛡️ **能量屏障区** | 第一回合所有单位获得 10 点护甲 | 前几回合的节奏改变 |
| 🌑 **暗物质区域** | 无法查看敌人意图 | 增加不确定性，考验经验判断 |
| 💾 **数据节点** | 每打出 3 张牌抽 1 张牌 | 鼓励快速出牌的引擎构筑 |
| 🔄 **时间扭曲** | 回合数上限 12 回合，超过则所有玩家即死 | DPS 检测，必须快速结束战斗 |
| 🧲 **磁场干扰** | 随机打乱玩家手牌顺序（视觉上），且每回合随机将一名角色换位 | 混乱中求生 |

#### 环境互动
- 某些卡牌可以**利用**环境效果（如"EMP 反转"在 EMP 干扰区中反而增强协议栈）
- 某些遗物可以**免疫**特定环境效果
- 多人合作时需要根据环境调整队伍站位和出牌策略

### ★6. 敌人适应性 AI (Adaptive Enemy AI)

敌人不再是固定意图循环的木桩，它们会**学习你的打法**。

#### 适应维度

| 维度 | 敌人响应 |
|------|---------|
| 玩家频繁使用攻击牌 | 敌人更频繁使用护甲/格挡意图 |
| 玩家频繁叠甲 | 敌人切换为穿甲/多段攻击 |
| 玩家集中攻击后排敌人 | 后排敌人获得"警惕"buff (闪避率+) |
| 玩家频繁使用入侵 | 敌人使用"防火墙"清除入侵进度 |
| 玩家总在前排 | 敌人使用更多 AOE/后排定点攻击 |
| 玩家总在后排 | 敌人使用"冲锋"直接攻击后排 |

#### 实现方式
- **非跨局适应**: 仅在**当前这一局游戏**中追踪玩家行为倾向
- 每场战斗结束后更新"玩家行为档案" (PlayerBehaviorProfile)
- 后续战斗中敌人 AI 的意图选择会参考这个档案，增加对应的反制意图权重
- **Boss 战适应更强**: Boss 会在战斗中**实时调整**意图（每 2-3 回合评估一次）
- 多人模式下，AI 会分析**整个队伍**的行为倾向

#### 平衡考量
- 适应性有**上限** (权重最多偏移 30%)，不会完全让某种策略无效
- 玩家可以通过**故意混合策略**来"欺骗" AI（增加博弈深度）
- 提供视觉反馈: 敌人"学习"时头顶出现扫描光效，让玩家感知到 AI 在适应

## 二、多人模式架构

### 2.1 多人模式总览

| 模式 | 人数 | 描述 |
|------|------|------|
| **单人模式** | 1 | 经典 Roguelike 体验，可带 AI 队友(可选) |
| **合作 PvE** | 2-4 | 共同探索巴别塔，共享地图，独立牌组 |
| **PvP 对战** | 2v2 / 1v1 | 竞技对抗，前后排站位策略对决 |

### 2.2 合作 PvE 模式设计

**参考杀戮尖塔2但做出差异化**:

| 维度 | 杀戮尖塔 2 | 我们的设计 |
|------|-----------|-----------|
| 站位 | 无 | ✅ 前后排系统，有仇恨/坦克机制 |
| 攻击目标 | 敌人总是全体攻击 | 敌人有**单体/群体/定位**攻击，前排承受更多 |
| 玩家互动 | 同时出牌，弱互动 | ✅ 协同链系统，强配合感 |
| 回合模式 | 同时行动 | **先同时规划，后按速度序列结算** |
| 角色差异化 | 各自独立 | ✅ 前后排定位 + 植入体 = 角色扮演感 |

#### 回合流程 (合作 PvE)

```
[玩家阶段 - 同时规划]
  │
  ├── 所有玩家同时查看手牌和敌人意图
  ├── 各自规划卡牌使用顺序 (可与队友语音沟通)
  ├── 可以在规划阶段调整前后排站位
  └── 全部玩家"确认" / 超时自动确认
  │
  ▼
[结算阶段 - 按速度序列]
  │
  ├── 1. 所有"快速"卡牌先结算 (标记为 Quick 的牌)
  ├── 2. 按玩家速度值从高到低逐个结算普通卡牌
  ├── 3. 链接效果在两张链接牌都打出后立刻触发
  └── 4. 所有"延迟"效果结算
  │
  ▼
[敌人阶段]
  │
  ├── 敌人按速度序列执行意图
  ├── 单体攻击 → 目标选择遵循仇恨规则
  ├── AOE → 按范围类型影响对应位置
  └── 适应性 AI 更新行为记录
  │
  ▼
[回合结束]
  ├── 环境效果结算
  ├── 状态效果 tick
  └── 进入下一回合
```

#### 多人专属内容
- 🔗 **链接卡**: 约占总卡池的 15%, 每个职业有 8-10 张链接卡
- 🤝 **合作遗物**: 效果影响全队的遗物（"量子纠缠器: 任一玩家获得护甲时,其他玩家获得2点护甲"）
- 🧪 **投掷药水**: 某些药水可以"投掷"给队友使用
- 🏕️ **休息点合作选项**: "修理队友装备"(帮队友升级一张牌), "守夜"(自己不回复HP,但队友回复量+50%)
- ⚰️ **倒下与复活**: 玩家 HP 归零后进入"濒死"状态(3回合)，队友可使用治疗将其拉起；3回合内无人救援则本场战斗退出，战后以 1 HP 复活

### 2.3 PvP 对战模式设计

#### 模式一: 1v1 对决
- 双方各选一个职业，使用**预构筑牌组**或**Ban/Pick 选牌**
- 战斗中有前后排但只有 1 人，前排=更高伤害但受到更多攻击; 后排=安全但输出低
- 双方同时规划出牌，同时结算
- 先将对手 HP 打到 0 者胜

#### 模式二: 2v2 团队对抗
- 每队 2 人，有前后排站位
- 更强调前后排的战术价值（坦克+输出 vs 双输出 等阵容选择）
- 协同链在队伍内可用
- 可攻击对方任一玩家（但攻击后排有伤害衰减，除非对方前排被击倒）

#### PvP 平衡考量
- PvP 和 PvE 使用**独立数值层** (PvP 卡牌效果可单独调整)
- 入侵系统在 PvP 中改为 "干扰" (暂时禁用对手下一次打出的牌，而非控制)
- 环境效果在 PvP 中随机轮换，双方受同样影响

### 2.4 网络架构

```
┌─────────────────────────────────────────────────┐
│                网络架构 (P2P + 可选中继)           │
│                                                   │
│  主机 (Host)                                      │
│  ├── 权威服务器 (Authoritative Server)            │
│  │   ├── 战斗状态同步                              │
│  │   ├── 种子随机数管理                            │
│  │   ├── 防作弊校验                                │
│  │   └── 存档管理                                  │
│  │                                                │
│  └── Steam P2P (via GodotSteam)                  │
│      ├── Lobby 系统 (房间创建/加入)                │
│      ├── NAT 穿透                                 │
│      └── Steam Relay 备选                         │
│                                                   │
│  客户端 (Client)                                  │
│  ├── 输入预测 (卡牌选择/站位切换)                   │
│  ├── 状态插值 (动画平滑)                           │

└─────────────────────────────────────────────────┘
```

**技术选型**:
- **Steam P2P**: 通过 GodotSteam 插件实现，无需自建服务器
- **主机权威模型**: Host 负责所有游戏逻辑运算，客户端只发送输入、接收状态
- **卡牌游戏友好**: 回合制 → 延迟容忍度高，不需要帧同步
- **状态同步**: 每次结算后同步完整战斗状态快照 (JSON 序列化)
- **种子随机**: Host 生成种子并分发，所有随机结果可验证

## 八、技术注意事项 (更新)

### 8.1 多人网络
- 回合制卡牌游戏延迟容忍度高，P2P 完全够用
- 关键: 状态同步的一致性，使用种子随机 + 状态校验
- 考虑 Steam Relay 作为 NAT 穿透失败的备选方案
- PvP 需要更严格的输入验证和反作弊

### 8.2 性能
- 多人模式下 4 名玩家 + 环境效果 + 链接特效 → 注意粒子和 UI 性能
- 使用对象池管理所有可复用的 UI 元素和特效
- 状态同步数据量控制: 只同步差异 (delta)，不是每帧全量

### 8.3 平衡性
- 单人/多人使用同一套数据但**数值乘数不同** (多人敌人 HP 按人数缩放)
- PvP 使用独立数值覆盖层
- 植入体是最大的平衡性挑战，需要大量测试
- 适应性 AI 的权重需要精调，避免玩家感觉"被针对"或"太弱"

### 8.4 存档系统 (更新)
- 单人存档: 本地 JSON + 校验
- 多人存档: Host 端管理, Steam 云存档同步
- PvP 记录: 对战录像 (输入流 + 种子 = 可完美回放)
