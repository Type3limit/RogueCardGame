## 核心设计哲学的问题：叠加 vs 融合

这份设计有 6 个"创新机制"，但它们之间的关系更像是**平行叠加**而非**有机融合**。好的游戏系统应该是"每个子系统的存在让其他系统变得更有趣"，而不是"每个子系统独立增加一层复杂度"。

来做个检验——如果拿掉某个系统，其他系统会变差吗？

| 拿掉的系统 | 其他系统受影响吗？ |
|------------|-------------------|
| 前后排站位 | 协同链、入侵、植入体、环境——全部可以脱离站位独立运作 |
| 协同链 | 站位、入侵、植入体、环境——完全不受影响 |
| 入侵/改造 | 只有黑客职业受影响，其他系统无感 |
| 植入体 | 遗物可以完全替代其角色 |
| 动态环境 | 删掉后战斗体验基本不变 |
| 适应性 AI | 删掉后反而可能更好玩 |

**这说明这些系统之间的耦合度太低。** 对比杀戮尖塔：能量、卡牌费用、抽牌数量这三个系统是深度纠缠的——你不可能拿掉任何一个而不摧毁整个游戏。这才是好的系统设计。

**建议：** 选 2-3 个机制，让它们深度咬合。比如如果选"站位 + 协同链"，那协同链的触发就应该和站位强绑定（前后排不同的链接效果、跨排链接 vs 同排链接的差异），而不是两个平行系统各自运作。

## 逐机制的设计评价

### 前后排站位：设计最成熟，但规则偏"安全"

站位系统的规则很完整，但设计倾向非常保守——它本质上是 MMORPG 仇恨系统的卡牌化。这不是坏事，但它没有充分利用卡牌游戏的特性。

**缺失的设计空间：**
- 站位切换本身没有成本与收益的张力。每回合免费切一次，太廉价了。如果切换需要消耗能量或者一张卡，玩家才会真正感受到"我选择待在前排"是一个有意义的决定。
- 前排近战满额、后排近战 -25%、后排远程 +15%——这些数值修正太线性了，没有创造出"不得不换位"的紧迫感。对比 Darkest Dungeon：某些技能只能在特定位置释放，位置错了就是废卡，这才逼迫玩家认真对待位置管理。
- 敌人的前后排"遮挡"规则没有进一步展开。如果前排敌人不仅遮挡攻击，还会对穿透行为做出反应（比如反击），就多了一层博弈。

**关键问题：** 站位系统目前对玩家的决策压力不够。坦克站前排、法师站后排，99% 的时间这就是最优解，换位是低频例外事件。好的站位系统应该让"该不该换位"成为每回合都要思考的问题。

### 协同链：最有创意，但有根本性的体验风险

协同链是整个设计中最吸引我的机制，它确实填补了"多人卡牌游戏中玩家之间缺乏有意义互动"的空白。但有几个设计层面的问题：

**问题一：信息不对称造成的沟通负担**

链接卡要求玩家知道队友手里有什么、打算打什么。但卡牌游戏的魅力之一是"在不确定中做决策"。如果高效配合要求完全透明的沟通（"我手里有标记牌，你有攻击牌吗？你准备打哪个？"），那每个回合都会变成漫长的协商，节奏会崩。

**对比**：好的合作桌游（如 Hanabi）通过限制沟通来制造张力。建议考虑：链接卡的触发不要求精确协调，而是基于"行为类别"的宽松匹配——你打了攻击，队友也打了攻击，就触发。不需要对准同一个目标。

**问题二："没有队友响应也有基础效果"削弱了机制的存在感**

文档里强调"链接卡即使没有队友也不是废卡"。这个出发点是对的，但如果基础效果太可用，玩家就没有动力去协调。如果基础效果太弱，单人模式就难受。这是一个两难。

**更好的思路：** 不要给链接卡"降级版基础效果"。让链接卡在单人模式中根本不出现在卡池里，用对应的非链接版本替代。这样多人模式和单人模式是两套不同的游戏体验，各自完整，而不是一个互相将就的折中。

**问题三：15% 卡池占比太高**

15% 意味着一手 5 张牌平均有 0.75 张链接卡。在 4 人合作中，每个回合有 3 张链接卡在场上等待响应。信息处理量太大了——你不仅要管自己的手牌，还要看 3 个队友谁打了链接卡、你能不能响应。这不是策略深度，是信息过载。

**建议：** 5-8% 就够了。链接卡应该是"这局 build 的高光时刻"，不是每回合的日常操作。

### 入侵/改造：设计方向好，但和核心循环脱节

入侵（控制敌人）和改造（双货币升级）在主题上很贴合科幻设定，但在玩法上存在一个根本问题：

**入侵是一条"非暴力路线"，但游戏的所有其他系统都在奖励暴力。**

站位系统的核心是"谁挡刀、谁输出"——这是暴力框架。协同链的主要类型是攻击链、增幅链——暴力框架。四个职业中三个的核心机制是伤害相关的——暴力框架。在这个环境下，花好几回合累积入侵进度去控制一个小怪 1-2 回合，几乎永远不如直接打死它。

**入侵有价值的唯一场景**是：控制的敌人强到足以改变战局（比如控制一个高伤敌人让它帮你打 Boss 护卫）。但文档限定了 Boss 不可入侵、只能控制 1-2 回合——这极大削弱了入侵的战略价值。

**改造的问题**是双货币的认知成本。金币和零件之间的决策空间有多大？如果两者用途明确分开（金币买成品、零件自定义），那就不存在"我该花哪种"的有趣决策——你在商店用金币，在改造站用零件，完事。真正有趣的双货币系统（如 FTL 的燃料/弹药/无人机零件）是你经常需要在两者之间做痛苦取舍。

### 植入体：设计理念很好，和遗物的区分需要更锐利

"遗物是加法，植入体是乘法"——这个哲学我很认同。Hades 2 的"月之恩赐"系统做的就是类似的事。

但看具体设计，有些植入体其实就是遗物：

- "纳米再生体：最大 HP -20%，每回合回复 3 HP"——这就是杀戮尖塔的"燃烧之血"遗物（打法不同的被动效果）
- "相位装甲：护甲保留 50%"——杀戮尖塔的"心灵绽放"遗物

**真正做到"乘法"的植入体应该改变你打牌的方式，而不只是改变数值。** 好的例子是文档中的"狂战核心"（取消超载，改为连续攻击递增）——这真正改变了你选什么卡、打牌的顺序。差的例子是大部分神经和躯体插槽的选项——它们只是数值调整。

**建议：** 砍掉所有"数值修正型"植入体，只保留"规则改变型"。宁可少，也要每个都足够震撼。

### 动态环境：好的调味料，但部分设计破坏了游戏核心乐趣

环境效果分两类：
1. **增加决策维度的**（好）：等离子风暴（攻击+3 技能+1费）让你重新评估手牌价值；数据节点（每 3 张牌抽 1 张）奖励特定 build。
2. **剥夺玩家能力的**（危险）：暗物质区域（无法查看敌人意图）、磁场干扰（随机打乱手牌 + 随机换位）。

第二类是很糟糕的设计。卡牌游戏的核心乐趣是"基于信息做出最优决策"。拿走信息（看不到意图）或者引入纯随机（随机换位）不是"增加挑战"，是在破坏玩家的代理感（agency）。

**对比**：杀戮尖塔中 Time Eater 的设计——它惩罚你打太多牌，但你完全知道这个规则、可以围绕它做策略调整。这是好的约束。而"随机打乱手牌"没有对策，只有运气。

**建议：** 环境效果应该只做"改变规则"类型（让玩家适应新规则），不做"剥夺信息/引入随机"类型。

### 适应性 AI：方向错了

这是我对整个设计中意见最大的一点。

**卡牌 Roguelike 的敌人不应该适应玩家。应该反过来——玩家去适应敌人。**

杀戮尖塔之所以耐玩，是因为你知道每个敌人的行为模式，核心乐趣是"针对已知的敌人行为构筑最优牌组"。如果敌人会适应你，那构筑的意义就被削弱了——你精心打造的攻击流 build 会被敌人"学会"然后针对，这不会让人觉得"哇敌人好聪明"，只会觉得"我的 build 被惩罚了"。

**更严重的问题**：在多人模式下，4 个玩家的行为倾向很难形成一致的"档案"。一个人全攻击、一个人全防御，AI 该怎么适应？适应哪个？文档说"分析整个队伍"，但这会导致 AI 行为变得不可预测且看起来随机，恰好和"让玩家读懂敌人"的设计目标相矛盾。

**替代方案：** 如果想让敌人更有趣，做**更丰富的固定行为模式**（比如敌人有 3 套行为循环根据当前 HP 阶段切换，类似 Boss 的多阶段设计），而不是动态适应。

## 回合流程设计的隐患

"同时规划 → 按速度结算"的回合制对多人合作有一个重大体验问题：

**等待时间。**

4 个玩家同时规划，最慢的那个决定了所有人的等待时间。而协同链还要求你规划前和队友沟通。这意味着每个回合的实际时间 = 沟通时间 + 最慢玩家的思考时间。一个回合可能 2-3 分钟。一场 12 回合的战斗就是 30 分钟。一次完整的 3 幕 run 可能 4-5 小时。

**对比**杀戮尖塔 2 的做法：玩家完全同时行动，不需要等，也很少需要协调。这牺牲了协同深度但保证了节奏。

你的设计想要深度协同又要多人——必须在协同深度和节奏之间做出取舍。不能两个都要。

## PvP 的根本矛盾

PvE Roguelike 的卡牌设计和 PvP 对战的卡牌设计有根本性的冲突：

- PvE 鼓励 build 极端化（全攻击、全防御、无限循环）——极端 build 带来的爽感是 Roguelike 的核心
- PvP 要求 build 均衡——极端 build 在 PvP 中要么 broken 要么 useless

文档提到"独立数值层"，但问题不在数值，在于**卡牌效果的设计逻辑不同**。一张"每回合复制手中所有攻击牌"的卡在 PvE 中是 build 的巅峰时刻，在 PvP 中是毁灭平衡的噩梦。数值调整解决不了效果层面的问题。

**建议：** PvP 如果要做，应该用完全独立的卡池（像 Legends of Runeterra 和 PvE 模式用不同的卡），而不是同一张卡换个数字。

## 总结：这个设计最需要的是做减法

| 维度 | 现状 | 建议 |
|------|------|------|
| 创新机制数量 | 6 个平行系统 | 选 2 个深度融合 |
| 职业数量 | 4 个 | 设计合理，保持 |
| 货币/资源种类 | 金币 + 零件 + 能量 + 各职业资源 | 砍零件，入侵改为纯卡牌机制 |
| 环境效果 | 10 种 | 砍掉"剥夺信息"类，保留 5 种"规则改变"类 |
| 敌人 AI | 适应性 | 改为多阶段固定行为 |
| 链接卡占比 | 15% | 5-8% |
| PvP | 1v1 + 2v2 + 匹配段位 | 要么做独立卡池的认真 PvP，要么完全不做 |
| 站位切换 | 每回合免费一次 | 加成本（耗能量或卡牌） |

**如果只能保留两个创新机制，我选"前后排站位"和"植入体（仅规则改变型）"。** 前者定义了战斗的空间维度，后者定义了构筑的深度维度。两者结合（站位影响植入体效果、植入体改变站位规则）可以形成紧密咬合的系统，而不是互不相干的功能堆叠。
