# RogueCardGame 设计备忘

> 此文档仅保留已纳入 v3 设计的内容。权威设计文档：[core-gameplay-design.md](core-gameplay-design.md)

## 已采纳的设计（已纳入 v3）

### 1. 位置反应型卡牌 ✅
同一张牌在前后排有不同效果。零新系统，直接强化站位核心轴。
- 实现：CardEffectData 的 FrontEffects / BackEffects 字段
- 首批每职业 1 张标志性位置反应卡

### 2. 原型核心系统 ✅
前 3 回合观察玩家行为后自动激活对应 buff，解决首幕无 build identity 问题。
- 三种协议：前压协议(近战)、战术协议(换位)、远程压制协议(远程)
- 系统根据行为自动激活，无需玩家开局盲选

### 3. 卡牌分支升级 ✅
同一张卡 A/B 两条升级路线，替代植入体固件升级成为中后期成长轴。
- 实现：CardUpgradeData 支持 upgradeA / upgradeB 分支
- 与位置反应型卡牌互补：升级可以强化前排版或后排版

### 4. 精英战场规则 ✅
每个精英战一条 twist，一句话说清楚。与现有敌人/环境/站位框架天然兼容。

### 5. 敌人站位关键词 ✅
首批两个：锁定（不换位受额外伤害）、破阵（削掩护）。
- 必须在敌人意图 UI 里一眼可见

### 6. 条件远程收益 ✅
每回合第一张远程牌 +20%，而非固定 +20%。削弱后排无脑收益。

## 设计原则

1. **不回到 v2 的"平行堆系统"**：新内容挂在现有骨架上（站位、植入体、敌人、奖励节奏）
2. **优先增加"可规划的张力"**：战斗内高信息（敌人意图可读），战斗外允许不确定性
3. **变化来自规则、情境和代价**：每个新内容问"它改变了玩家这回合的操作顺序吗？"
