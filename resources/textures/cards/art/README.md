# Card Art Pipeline

Drop per-card illustrations under this folder and the UI will pick them up automatically.

Important:

- Files here should be illustration layers for the art stage, not full card mockups.
- Do not bake card borders, title text, rarity labels, or full-frame UI into these assets.
- Placeholder SVGs generated by the repo script are intentionally transparent illustration-only layers.

Supported explicit JSON field:

```json
"artPath": "vanguard/vanguard_strike.png"
```

Also supported:

- Full Godot resource paths such as `res://resources/textures/cards/art/vanguard/vanguard_strike.webp`
- Relative names such as `vanguard_strike` or `vanguard/strike`

Automatic discovery order when `artPath` is omitted:

1. `res://resources/textures/cards/art/<class>/<card_id>.<ext>`
2. `res://resources/textures/cards/art/<card_id>.<ext>`
3. `res://resources/textures/cards/art/<class>/<nameEn_slug>.<ext>`
4. `res://resources/textures/cards/art/<nameEn_slug>.<ext>`
5. Fallback to generic type art

Supported extensions:

- `.png`
- `.webp`
- `.jpg`
- `.jpeg`
- `.svg`

Recommended convention:

- Use lowercase snake_case file names
- Prefer `<card_id>.png` under the class folder
- Keep source art portrait-oriented for best results in the current poster frame
- Keep the outer background transparent or very subdued so the runtime card frame stays readable

Examples:

- `res://resources/textures/cards/art/vanguard/vanguard_strike.png`
- `res://resources/textures/cards/art/psion/psi_bolt.webp`
- `res://resources/textures/cards/art/netrunner/ghost_probe.png`
